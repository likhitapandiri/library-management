Prisma CLI:
Reads schema.prisma
Compares it with previous migrations
Generates SQL
Opens a database connection
Executes SQL
Then How Does Generate Work?

at it also runs npx prisma generate
No database is required.No connection pool is required.No MySQL query is required.

It simply:
Read schema.prisma
       ↓
Parse models
       ↓
Generate JavaScript/TypeScript code
       ↓
Write files to disk

Example:
model Book {
  id Int @id
}

becomes generated code similar to:

class PrismaClient {

  book = {
    findMany(){},
    create(){},
    update(){}
  }

}

and saves it into:node_modules/@prisma/client(or the custom output folder).
